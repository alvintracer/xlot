

### 1. 중계 서버(Relay Server)란 무엇이고, 왜 필요한가요?

이 서버는 대표님의 프론트엔드(React 앱)와 업비트(Upbit) API 서버 사이에서 통신을 중계하는 역할을 합니다. 

* **보안 (가장 중요):** 업비트 API를 호출하려면 `Secret Key`가 필요한데, 이걸 프론트엔드(브라우저)에서 직접 사용하면 해킹의 위험이 있습니다. 중계 서버가 이 키를 받아 **안전하게 서명(JWT 토큰 생성)**만 해서 업비트로 보내줍니다.
* **업비트 까탈스러움 해결 (우리가 고생한 부분):** 업비트는 데이터를 보낼 때 글자 순서 하나, 띄어쓰기 하나, 심지어 `null` 값 포함 여부까지 정확히 따져서 암호화(Hash)해야 합니다. 프론트엔드에서 이걸 맞추기 어려우므로, 중계 서버에서 **업비트 공식 규격에 맞춰 데이터를 예쁘게 포장(`encodeQuery`)**해주는 역할을 합니다.

### 2. 서버 파일 위치 및 핵심 코드 (`server.js`)

서버 코드는 클라우드 서버(또는 로컬 PC)의 작업 폴더 안에 **`server.js`** 라는 이름의 파일로 존재합니다. (Node.js 기반의 Express 프레임워크를 사용했습니다.)

**[가장 건드리면 안 되는 핵심 로직]**
서버 코드 상단의 `encodeQuery` 함수는 우리가 수많은 에러(`invalid_query_payload`)를 뚫고 찾아낸 정답입니다. 이 부분은 절대 건드리지 마세요!
```javascript
// null, undefined를 걸러내고, 업비트 규격에 맞춰 URL 인코딩하는 핵심 함수
function encodeQuery(params) {
    const keys = Object.keys(params)
        .filter(k => params[k] !== undefined && params[k] !== null && params[k] !== '')
        .sort();

    return keys
        .map(k => `${encodeURIComponent(k)}=${encodeURIComponent(String(params[k]))}`)
        .join('&');
}
```

### 3. 서버를 켜고, 끄고, 재시작하는 방법 (PM2 사용법)

우리는 서버가 꺼지지 않고 24시간 백그라운드에서 돌아가도록 **`pm2`** 라는 프로그램을 사용하고 있습니다. 터미널(명령 프롬프트)에서 아래 명령어들을 사용하세요.

* **서버 처음 켜기 (또는 다시 켜기):**
  ```bash
  pm2 start server.js
  ```
* **서버 코드 수정 후 적용하기 (재시작 - 가장 많이 씁니다):**
  ```bash
  pm2 restart server
  ```
* **서버 작동 멈추기:**
  ```bash
  pm2 stop server
  ```
* **서버 목록에서 아예 지우기 (초기화 필요할 때):**
  ```bash
  pm2 delete server
  ```
* **👀 서버가 뱉어내는 로그(에러 메시지 등) 실시간 확인하기:**
  ```bash
  pm2 log
  ```
  *(로그 화면에서 빠져나오려면 `Ctrl + C`를 누르시면 됩니다.)*

### 4. 서버 코드를 수정하는 방법 (`nano` 에디터)

새로운 거래소(예: 빗썸, 바이낸스)를 추가하거나 코드를 고쳐야 할 때는 터미널에서 텍스트 에디터를 엽니다.

1. **파일 열기:**
   ```bash
   nano server.js
   ```
2. **코드 수정:** 키보드 방향키로 이동하며 코드를 지우거나 붙여넣기 합니다. (마우스 클릭은 안 먹히니 방향키를 쓰세요!)
3. **저장하고 나가기 (순서대로 누르세요):**
   * `Ctrl + O` (알파벳 O입니다. 저장 모드)
   * `Enter` (파일명 확인 후 엔터)
   * `Ctrl + X` (에디터 종료)
4. **수정 후에는 무조건 재시작!**
   ```bash
   pm2 restart server
   ```

### 💡 요약 팁

앞으로 프론트엔드에서 업비트로 무언가 새로운 요청(새로운 API 연동 등)을 만들어야 한다면, 프론트엔드에서 직접 업비트로 쏘지 마시고, **반드시 `server.js`에 새로운 라우트(`app.post('/upbit/새로운기능', ... )`)를 뚫어서 경유**하도록 만드셔야 합니다.

이 매뉴얼만 있으면 언제든 서버를 통제하실 수 있을 겁니다! 더 궁금한 점이 생기면 언제든 말씀해 주세요.
